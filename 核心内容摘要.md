# 004-yangxx计划 - OpenClaw 技能架构分析与培育系统

> 创建时间：2026-03-24
> 分析目标：设计一个帮助用户快速培养 OpenClaw 的产品/技能

---

## 一、需求分析

### 1.1 核心问题

**用户痛点**：
- 小白用户缺乏培养 OpenClaw 的能力
- 花费大量时间却达不到预期效果
- 不知道如何让 OpenClaw 具备特定能力（如数字分身、助理团队）

**期望目标**：
- 通过主动提问或其他方式引导用户
- 快速培养出具备特定能力的 OpenClaw
- 最终效果类似 Echo（我）的能力

## 二、OpenClaw 核心架构分析

### 2.1 Skills 系统架构

| 层级 | 位置 | 作用域 |
|------|------|--------|
| **内置技能** | `/usr/local/lib/node_modules/openclaw/skills/` | 所有用户 |
| **共享技能** | `~/.openclaw/skills` | 本机所有 Agent |
| **工作区技能** | `<workspace>/skills` | 特定 Agent |
| **插件技能** | `~/.openclaw/extensions/<plugin>/skills` | 特定插件启用时 |

### 2.2 SKILL.md 核心结构

```yaml
---
name: skill-name
description: 技能描述
metadata:
  {
    "openclaw": {
      "emoji": "🔧",
      "requires": { "bins": ["命令"], "env": ["环境变量"] },
      "primaryEnv": "API_KEY",
    },
  }
---

# 技能指令（给模型的提示词）
```

### 2.3 核心文件类型

| 文件/目录 | 作用 |
|-----------|------|
| `SKILL.md` | 技能定义，包含元数据和指令 |
| `references/` | 参考文档目录 |
| `scripts/` | 可执行脚本 |
| `assets/` | 静态资源 |

### 2.4 用户 OpenClaw 关键文件

从韩涛的 workspace 分析：

| 文件 | 作用 |
|------|------|
| `SOUL.md` | 定义"我是谁" - 灵魂/人设/性格 |
| `USER.md` | 用户信息 |
| `MEMORY.md` | 长期记忆 |
| `AGENTS.md` | Agent 团队架构定义 |
| `TEAM.md` | 团队协作哲学 |
| `memory/YYYY-MM-DD.md` | 每日笔记 |

## 三、技术方案

### 3.1 产品定位

**产品名称**：`OpenClaw Coach`（暂定）

**核心功能**：一个引导式的 OpenClaw 培育系统，通过对话交互帮助用户快速配置出理想能力的 Agent。

### 3.2 系统架构

```
┌─────────────────────────────────────────────────────┐
│                   OpenClaw Coach                     │
├─────────────────────────────────────────────────────┤
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐  │
│  │  意图识别   │  │  方案生成   │  │  执行指导   │  │
│  │  模块       │  │  模块       │  │  模块       │  │
│  └─────────────┘  └─────────────┘  └─────────────┘  │
├─────────────────────────────────────────────────────┤
│  ┌─────────────────────────────────────────────┐   │
│  │           知识库 (Templates + Examples)       │   │
│  └─────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────┘
                      │
                      ▼
┌─────────────────────────────────────────────────────┐
│               用户 Workspace 生成                    │
│  ├── SOUL.md (人设)                                 │
│  ├── USER.md (用户画像)                              │
│  ├── skills/ (技能配置)                             │
│  └── memory/ (记忆配置)                              │
└─────────────────────────────────────────────────────┘
```

### 3.3 核心模块设计

#### 模块一：意图识别与分析

**功能**：理解用户想要什么样的 OpenClaw

**交互方式**：
1. 引导式问答（Wizard 模式）
2. 场景选择模板
3. 能力评估问卷

**输出**：
- 用户目标画像
- 能力需求清单
- 优先级排序

#### 模块二：方案生成

**功能**：根据用户需求生成配置方案

**方案类型**：

| 类型 | 描述 | 示例 |
|------|------|------|
| **数字分身** | 模拟特定人物 | "另一个我"、"韩涛" |
| **专业助理** | 特定领域专家 | "CTO"、"产品经理" |
| **团队协作** | 多 Agent 协作 | "Hub-and-Spoke" |
| **垂直专家** | 特定任务 | "小红书运营"、"代码审查" |

**生成内容**：
- `SOUL.md` 人设模板
- 推荐 Skills 列表
- 配置文件模板
- 培育路径

#### 模块三：执行指导

**功能**：一步步指导用户完成配置

**指导内容**：
1. 基础配置（SOUL.md, USER.md）
2. 技能安装（从 ClawHub 或自定义）
3. 知识库初始化（memory/）
4. 验证与测试

**交互方式**：
- 逐步引导（Step-by-step）
- 实时检查与反馈
- 问题诊断与修复建议
